# 养龙虾观点摘录（统一整理版）

> 来源：
> - 声东击西对谈（Justin / Junyu / 徐涛）
> - 枫言枫语对谈（Justin / 自力）
>
> 用途：
> - 为 `NexAgent` 的产品叙事、对外分享和内部讨论提供素材
> - 从“养龙虾 / OpenClaw / Vibe Coding”实践中提炼稳定观点，而不是保留零散摘录

---

## 一页总结

这两场对谈反复指向的是同一件事：

1. “养一个自己的 agent”最重要的价值，不是复制一个功能集合，而是亲自踩坑，获得创造者视角。
2. Agent 软件和传统软件的差别，不在于“多接了一个模型”，而在于 agent 是否已经成为系统的核心。
3. 真正有启发的能力，不只是会聊天，而是会把问题转成代码、流程、skills，甚至自我修改。
4. 长期价值不在“平均能力”，而在专属方法、个人品位、差异化流程，以及“一人一个 App”的可能性。
5. 现实边界也很明确：模型能力仍有限，安全边界难，token 成本高，距离成熟商业化还有距离。

---

## 一、为什么大家都想“养一个自己的龙虾”

### 1. 为了亲自踩坑，而不是旁观理解

Justin 最核心的判断是：很多东西不亲手做一遍，根本理解不到位。

> “我的朋友们都做了一个自己的 agent。”
>
> “我做的目的主要还是为了学习。我不做一个的话，我可能没有办法踩过他一样踩过的坑。”

这里最重要的不是“做出一个和别人一样的东西”，而是：

- 你会真正理解 agent 接不同模型、不同工具、不同通道时到底有多少细节
- 你会看到哪些地方是 prompt 能解决的，哪些地方必须靠系统设计解决
- 你会从使用者视角，切换到创造者视角

### 2. 从“会用”变成“会造”

Justin 对这一点说得很透：

> “做的过程中呢，其实我是站在创造者的角度去想，我要往里面加什么东西，而不是说我通过一个纯使用者的角度去想，我要怎么用它，这两个角度是不太一样的。”

这也是“养龙虾”最有价值的地方：你不再只是体验 agent，而是开始思考 agent 为什么会这样工作，以及它下一步应该长成什么样。

### 3. 做，比想更能打开脑洞

Junyu 的经验很直接：

> “但是你试一下做一个简单的东西，你还是会打开很多新的思路的。”
>
> “大部分的东西其实你光思考是思考不出来，做了之后你突然你的脑洞就被打开了。所以做很重要，所以要一直去用 AI。”

这里的关键词不是“正确规划”，而是“通过实践进入新的认知层”。

## 二、Agent 软件和传统软件到底哪里不一样

### 1. Agent 不是外挂，而是核心

Justin 对 OpenClaw 和传统软件的区别有一个很准确的表述：

> “传统软件的核心依然是传统软件设计的思路。AI 在这里它是一个辅助的角色，它并不是一个核心的角色，但 OpenClaw 不一样。”
>
> “如果这个 agent 被拿掉了，那么这个软件就彻底不成立。”

这句话很关键。很多“AI 产品”本质上只是给原来软件多加了一个 AI 入口；但 agent 软件真正成立的前提，是 agent 本身已经成为系统中枢。

### 2. 从“预制 APP”到“现炒 APP”

这场对谈里最形象的比喻之一，就是“预制 APP”与“现炒 APP”。

> “以前的软件需要我们去手搓，有了 Vibe Coding 之后我们发现可以用 AI 帮我们手搓，现在我们发现可以让师傅现做了，都不需要预制了。”
>
> “哦，以前我们都是预制 APP，好吧，现炒 APP 现在。”

它揭示的是一个更大的转变：

- 传统软件：先定义功能，再写代码，再部署给用户
- Agent 软件：用户说一句话，系统现场理解、现场组合能力、现场产出结果

如果这条路成立，软件的基本形态就会发生变化。

### 3. 从“个性化配置”走向“专属系统”

自力提出的“一人一个 App”很有代表性：

> “这个我觉得可能是未来非常有吸引力的一个事情，一人一个 App。”

Justin 的补充也很重要：

> “专属，对，它是一个高级个性化，就是专属的东西在里面。”

这背后的转向是：

- 以前的软件追求的是“给更多人提供更多配置项”
- 未来的 agent 追求的是“围绕具体的人、具体的方法、具体的工作流生成专属能力”

## 三、最有冲击力的几个 Aha Moment

### 1. 代码变成了解决问题的通用语言

Junyu 的第一个顿悟，是看见 agent 在“做不对”之后，开始自己写代码解决问题。

> “一开始想让她做这个事情，她就做不对。然后我就发现他开始写代码。”
>
> “其实就是说你给了一个程序员，你让程序员去做各种职业是很有意思的。他其实会用写代码的方式去解决各种看起来不是软件工程的问题。”

这是一个很大的认知变化：代码不再只是工程师交付软件的工具，而开始成为 agent 处理各种任务的通用表达方式。

### 2. 它不只是执行，还会改自己

另一层震撼来自 agent 的自修改能力：

> “更神奇的是，你会看到他自己修改自己嘛，就是他自己修改自己的代码，经常把自己改坏。”

徐涛的 Aha Moment 也来自类似经历：

> “直到有一次，他其实是 memory 爆掉了……我跑到他的后端去看 memory 的时候，就发现原来他已经在后台写了那么多程序。”

这说明 agent 的行为已经不再只是“回答问题”，而是在内部形成了持续运作的程序化结构。

### 3. 它会给出你没预料到的能力

Justin 讲到这种“随机惊喜”时，用了一个非常好的比喻：

> “你以前给他 1+1=2，他一定等于 2，现在不是了，你可能写了一个 1+1，但是他突然会乘除法了，然后你也不知道谁教他的。”

这类体验会反复出现：

- 没特意做语音能力，但它突然能解析语音
- 没特意做图片能力，但它突然能识图
- 没特意教它某条工作流，但它会自己串起来

这正是 agent 软件和传统软件最不一样的地方之一：系统会出现超出开发者原始预期的能力组合。

### 4. Random Surprise 的价值，不只是好玩

Justin 后来甚至主动设计“随机惊喜”，目的不是单纯逗用户开心，而是为了打破人的思维惯性。

> “我想利用它的这一点，倒不完全是为了多巴胺的升起而下降，而是为了它能够刺激我，跳脱我原有的固有的思维方式。”

也就是说，agent 的一个独特价值，不只是替你做事，而是反过来推动你重新思考问题。

## 四、对 Agent 设计最有价值的启发

### 1. 记忆不该只记标签，更要记方法

Junyu 区分得很清楚：

> “ChatGPT 过去实现的长记忆，他只记住了一些标签或者是一些事件……但是他其实不知道你偏好的做事的方法。”
>
> “你是可以教它做事的方法，它可以把这个步骤给记下来，并且去反复地去使用。”

这意味着更有价值的记忆，不只是“你喜欢什么”，而是“你通常怎么做事情”。

### 2. Skills 的本质，是把工作流固化下来

Junyu 的实践总结非常接近现实团队带新人的过程：

> “这样子我不同的事情我就去让 agent 去做。他再去看我的流程，然后再去按照这个流程去执行，并且这个过程当中他还可以去改进这个流程。”

这也是 skill 系统最重要的意义：

- 它不是另一个花哨的 prompt 容器
- 它是把做事方法沉淀成可复用模块
- 它让 agent 有机会从一次次反馈中提炼稳定流程

### 3. 主动性未必来自复杂架构，可能只来自一个 timer

有一个特别值得记住的点是：

> “它就是每 30 分钟主动运行一次……一个定时器。对，就是这么简单一个事情。”

这提醒我们，很多“看起来像智能体主动性”的能力，并不一定需要极其复杂的系统设计；有时候只是一个简单但正确的机制被放进了合适的场景里。

### 4. 教 agent，越来越像教新人

这个比喻在两场对谈里都反复出现：

> “这就有点像你有个相对来讲比较聪明、比较靠谱的一个员工在帮你来做这个事情。你去 coach 他，你去 train 他的这个过程，确实我觉得跟原来做新人的入职是会有一点点像。”

未来的关键能力，也许不是“自己把事做完”，而是：

- 你能不能把做事方法讲清楚
- 你能不能判断结果好坏
- 你能不能持续校正和训练一个智能体

## 五、对 NexAgent 的直接启发

如果把上面的观点收束到 `NexAgent`，可以提炼出几条非常直接的设计原则。

### 1. Agent 应该长期在线，而不是只跑一轮

“养龙虾”真正成立的前提，是 agent 进入日常环境，长期存在，而不是每次都从零开始。

这对应到 `NexAgent` 的方向就是：

- 它应该在聊天应用里长期在线
- 它应该维护会话，而不是只看当前窗口
- 它应该有后台任务，而不是只能被动响应

### 2. 进化必须分层，不是什么都改代码

两场对谈都说明了一件事：agent 的成长不只是“能力变强”，而是学习事实、方法、工具和系统边界。

这和 `NexAgent` 当前的六层进化框架是对齐的：

- `SOUL`：身份、价值观、行为准则
- `USER`：用户画像和协作偏好
- `MEMORY`：长期环境与项目事实
- `SKILL`：可复用的方法和流程
- `TOOL`：确定性的可执行能力
- `CODE`：当上层无法解决问题时才进入源码升级

这会比“什么问题都直接改 prompt / 改代码”更稳定。

### 3. Skill 系统不该只是文档，而该成为运行时能力的一部分

OpenClaw 的启发之一，是 Markdown 形式的 skill 可以极大降低能力扩展的门槛。

对 `NexAgent` 来说，这意味着：

- skill 应该易于创建、读取、沉淀和复用
- skill 不只是写给人看的文档，而是 agent 运行时可以发现和调用的能力包
- skill 应该能承接“教方法”这件事

### 4. 简单的调度机制，也能带来显著主动性

“每 30 分钟跑一次”这种思路，对 `NexAgent` 的 `cron` 能力是非常直接的启发。

也就是说，agent 的主动性不一定先来自超复杂 planning，而可以先来自：

- 定时触发
- 后台执行
- 轻量上下文
- 与主会话隔离

### 5. 安全边界必须从一开始就承认是核心问题

Justin 对安全问题的提醒非常现实：

> “我觉得他觉得没关系，只不过是因为他还不知道它能干什么。”

对 `NexAgent` 来说，这意味着：

- tools、skills、code 不能混成一团
- 自动调用和显式授权要区分
- 长期在线 agent 的权限边界，比一次性 demo 更重要

### 6. 真正的差异化，不会来自“平均能力”

这几场对谈的最终结论之一其实很适合 `NexAgent`：

- 不要把目标设成“再做一个什么都能做一点的平均 agent”
- 要把重点放在长期记忆、专属方法、工作流沉淀和真实场景中的持续进化上

## 六、现实边界：什么是“龙虾”现在还做不到的

### 1. 模型能力依然有限

Justin 对这个边界看得很清楚：

> “前期你会发现，哇，小龙虾好牛逼啊，啥都能做……但你用久了之后就发现，模型现在的能力依然是有限。”

典型表现包括：

- 重复
- 跑偏
- 看起来会，细做不稳
- 复杂任务成本过高

### 2. AI 能做平均值，但未必做得出顶级表达

徐涛和 Justin 都强调了这一点：

> “AI 是做不到 top，做到 90 分以上。”
>
> “未来可能所有人都能做最大公约数。那么你的独特性在哪里？你的个性在哪里？”

这意味着：

- AI 会显著抬高平均线
- 但高水平表达、审美、判断和人格化风格仍然稀缺

### 3. 安全问题不是附属问题

尤其当 agent 进入消息、网络、文件、脚本和自动执行之后，安全边界会成为产品定义的一部分，而不是上线前补的一个 checklist。

### 4. 商业化受 token 成本明显制约

对谈里对这点也没有回避：

> “这一切是可行的，如果落到今天的现实，要做这样的软件，且要从里面回本的话，几乎不太可能。”

短期内，这类产品的约束非常现实：

- 小任务也可能触发大开销
- agent 容易把简单问题做成 deep research
- 长期运行意味着持续烧 token

## 七、给实践者的建议

### 1. 持续去用最好的 AI

Justin 的第一条建议很直接：

> “要想尽一切办法用上最好的 AI，跟上他们新出的东西是什么，最好是去用，因为用了才能更好的体会它。”

### 2. 基础能力在 AI 时代反而更重要

这也是一个反直觉但很重要的判断：

> “在 AI 这个时代好像 AI 看起来无所不能，但事实上基础会变得更重要。”

原因不是因为 AI 不强，而是你仍然需要判断：

- 它哪里对
- 它哪里错
- 它哪里只是看起来像对

### 3. 品位和独特性会越来越贵

Justin 对“最大公约数”的提醒，本质上是在说：

- 当通用能力被拉平，差异化会转移到判断力、审美、表达和取舍
- 未来更值钱的，不是“会不会生成”，而是“你知道什么值得生成”

### 4. 先做一个自己的，再谈抽象理解

这大概是整份材料最适合落地的一条建议：

- 先做一个很小的 agent
- 先让它进一个真实场景
- 先让它承担一点长期任务
- 在踩坑中理解真正的问题

## 八、可直接复用的表达

下面这些句子适合在分享、写作或介绍 `NexAgent` 时直接借用或转写。

### 适合讲“为什么自己做”

- “做一个自己的 agent，价值不在复制功能，而在亲自踩过那些别人踩过的坑。”
- “从使用者变成创造者之后，你思考的问题就完全不一样了。”
- “很多关于 agent 的理解，不是看出来的，是做出来的。”

### 适合讲“为什么 agent 软件不一样”

- “传统软件是预制的，agent 软件开始走向现炒。”
- “如果把 agent 拿掉，系统就不成立，这才说明 agent 是核心，而不是外挂。”
- “未来可能不是一个超级 App 服务所有人，而是更多‘一人一个 App’式的专属系统。”

### 适合讲“为什么 skill / memory 重要”

- “真正有价值的记忆，不只是记住你是谁，而是记住你通常怎么做事。”
- “Skill 的本质不是多一份 prompt，而是把方法沉淀成可复用工作流。”
- “教 agent，越来越像带一个新人：你要能讲清方法，也要能判断结果。”

### 适合讲“为什么还要保持清醒”

- “AI 会把平均线抬高，但不会自动给你顶级表达、个人品位和判断力。”
- “安全边界不是上线前补一层，而是长期在线 agent 的产品核心。”
- “这条路是成立的，但今天离低成本、稳态商业化还有距离。”

---

## 结论

“养龙虾”的价值，不只是让人感叹 AI 很神奇，而是逼着我们重新理解：

- 软件应该怎么被构建
- 人和 agent 应该如何协作
- 什么能力该沉淀成记忆、方法、工具或代码
- 什么才是 AI 时代真正稀缺的差异化

如果把这些材料用一句话收束，它更像是在说：

> 未来真正有价值的 agent，不是“什么都能做一点”的平均代理，而是能够长期在线、理解具体人、沉淀具体方法、在真实场景中持续进化的专属系统。
